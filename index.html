1. Configuration SPF (Sender Policy Framework)

Objectif: Définir quels serveurs peuvent envoyer des emails pour votre domaine

Google Workspace uniquement:

v=spf1 include:_spf.google.com ~all

Google Workspace + Instantly.ai (RECOMMANDÉ):

v=spf1 include:_spf.google.com include:_spf.instantly.ai ~all
⚠️ Limite critique: Maximum 10 DNS lookups par record SPF

Délai d'activation: 24-48 heures

2. Configuration DKIM (DomainKeys Identified Mail)

Objectif: Signature cryptographique pour authentifier vos emails

Étapes Google Workspace:

  1. Admin Console → Apps → Gmail → Authenticate email
  2. Générer nouvelle clé DKIM 2048 bits (obligatoire 2025)
  3. Copier le TXT record généré
  4. Ajouter dans DNS: google._domainkey.votredomaine.com
  5. Attendre 24-72h puis activer dans Admin Console
Best practice 2025: Utiliser uniquement des clés DKIM 2048-bit minimum

3. Configuration DMARC (Domain-based Message Authentication)

Objectif: Politique de traitement des emails échouant SPF/DKIM

Phase 1 - Monitoring (Semaines 1-4):

v=DMARC1; p=none; rua=mailto:dmarc@votredomaine.com; fo=1

→ Aucune action, collecte de données

Phase 2 - Quarantine (Semaines 5-8):

v=DMARC1; p=quarantine; pct=10; rua=mailto:dmarc@votredomaine.com

→ 10% des emails suspects en spam (augmenter progressivement)

Phase 3 - Reject (Après validation):

v=DMARC1; p=reject; rua=mailto:dmarc@votredomaine.com; pct=100

→ Rejette 100% des emails non conformes

🚨 CRITIQUE: Attendre 48h après SPF/DKIM avant d'activer DMARC. Commencer TOUJOURS par p=none

4. MX Records Configuration

Pour Google Workspace:

Priorité 1: ASPMX.L.GOOGLE.COM Priorité 5: ALT1.ASPMX.L.GOOGLE.COM Priorité 5: ALT2.ASPMX.L.GOOGLE.COM Priorité 10: ALT3.ASPMX.L.GOOGLE.COM Priorité 10: ALT4.ASPMX.L.GOOGLE.COM
